# Phase 4: Exam Intelligence

## Goal
Connect student knowledge with actual UPSC exam patterns.

## What gets built
### 1. PYQ reverse engineering
Build a structured past-year-question dataset.

Need:
- topic tagging
- concept tagging
- frequency analysis
- recurrence patterns

### 2. Topic priority engine
The system should estimate:
- which concepts are frequently tested
- which concepts are overdue
- which concepts the student is weak in

This lets myCELIA prioritize study intelligently.

### 3. Visual exam intelligence
Potential outputs:
- topic heatmaps
- priority labels
- recommended study order

## Why this phase matters
This phase ensures the product is not only personalized.
It is also exam-aware.

That combination is powerful:
- what the exam wants
- what the student knows
- where the student is weak

## Success criteria
Phase 4 is complete only when:
- PYQ data is usable and tagged
- concept priority can be calculated
- student study recommendations reflect real exam weight
